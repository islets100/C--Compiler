## 二、符合 finaltask 要求规范的实现代码

根据 `finaltask.md` 的要求，中间代码生成部分需要：
1. 遍历语法分析器生成的语法树
2. 访问语法树结点并调用中端代码
3. 最终输出 `.ll` 格式的 LLVM IR

## 四、单独测试 complie_ir 的终端指令

### 4.1 Windows 环境（使用 PowerShell）

#### 方法一：直接运行已编译的可执行文件

```powershell
# 进入 complie_ir 目录
cd complie_ir

# 直接运行 IRmain.exe（需要先编译）
# 注意：程序默认读取 ../Syntax/output/tree8.txt
# 修改 IRmain.cpp 第170行可以更改输入文件
.\IRmain.exe
```

#### 方法二：使用 CMake 编译并运行

```powershell
# 进入 complie_ir 目录
cd complie_ir

# 创建 build 目录（如果不存在）
if (-not (Test-Path build)) { New-Item -ItemType Directory -Path build }

# 进入 build 目录
cd build

# 运行 CMake 配置
cmake ..

# 编译
cmake --build .

# 运行（可执行文件名称根据 CMakeLists.txt 确定）
# 通常为 project1.exe 或 IRmain.exe
.\project1.exe

# 或者返回到 complie_ir 目录运行
cd ..
.\build\project1.exe
```

## 六、关键实现点总结

1. **语法树遍历**：从 Program 节点开始，递归访问所有语法树节点

3. **类型系统**：支持 int、float、bool、void 类型
4. **表达式求值**：按照运算符优先级实现所有表达式
5. **控制流**：正确处理 if-else、return 等控制语句
6. **函数处理**：支持函数定义、参数、调用
7. **浮点数支持**：完整的浮点数运算和比较支持（finaltask 之外的扩展）
